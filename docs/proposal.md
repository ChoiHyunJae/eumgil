## 핵심 철학

"노인을 수혜자가 아닌 지역 연결의 주체로"

수십 년간 동네에서 살아온 노인을 지역 전문가로 보고, 그 지식을 ICT로 아카이빙하여
새 입주민·관광객과 연결한다. 노인이 지역의 연결자가 되는 구조.

## 서비스 대상

| 구분 | 역할 | 주요 행동 |
|---|---|---|
| 동네 노인 (안내자) | 지역 전문가·연결 주체 | 동네 지식 등록, 새 주민과 동행 |
| 새 입주민 | 지역 탐색자 | 동네 정보 탐색, 안내자와 매칭 |
| 관광객 | 방문 탐색자 | 근거리 안내자 매칭, 지역 체험 |

## 핵심 기능 3가지 (확정)

### ① 동네 지식 아카이브 (노인 → 지역 전문가)
- 노인이 음성 또는 사진으로 지역 정보를 간편 등록
- 등록 가능 정보: 단골 가게, 조용한 산책로, 병원 가는 법, 계절 행사, 지역 역사 등
- 텍스트 입력 최소화 → 음성 녹음 + 사진만으로 등록 완료 (노인 친화적 UX)

### ② 근거리 동행 매칭 (노인 ↔ 새 입주민/관광객)
- 반경 500m~1km 이내 초근거리 매칭
- 단순 정보 전달이 아닌 '같이 걷기' 동행으로 실제 만남 연결
- 매칭 수락 후 만남 장소·시간 확인 단계를 거침 (수락 후 실제 만남 불발 문제 보완)
- 안내자 등록은 복지관·주민센터를 통한 사전 확인을 거침
- 동행 시작·종료 시 지정 연락처로 알림 발송
- 세대 구분 없이 누구든 안내자/탐방자로 참여 가능

### ③ 소모임 자동 전환
- 동행이 2~3회 반복되면 앱이 '정기 모임 만들어볼까요?' 자동 제안
- 노인이 직접 정기 모임을 개설하거나 참여 가능
- 사용자가 커뮤니티를 직접 만들려는 의도 없이도 반복 만남이 지역 기반 소모임으로 발전

## 기술 스택 (확정 — 변경 불가)

| 영역 | 기술 | 선택 이유 |
|---|---|---|
| 모바일 앱 (노인용) | **Flutter** | iOS·Android 동시 개발, 노인 친화 UI 구현 용이 |
| 백엔드 | Firebase (Firestore + Functions) | 실시간 위치 기반 매칭, 빠른 프로토타입 |
| 위치 기반 매칭 | Google Maps API | 반경 계산, 근거리 매칭 로직 |
| 음성 등록 | Google STT API | 노인 음성 → 텍스트 자동 변환 |
| 알림 발송 | 카카오 알림톡 API | 노인 친화적 (카카오톡 기반) |

> 플랫폼: 네이티브 앱(Flutter 기반 크로스플랫폼). Android 단일 플랫폼 아님 — iOS·Android 동시 지원이 신청서에 명시된 전제.

## 개발 일정 (총 12주, 확정)

| 단계 | 기간 | 주요 내용 | 산출물 |
|---|---|---|---|
| 1단계: 기획·설계 | 1~2주 | UI/UX 설계, DB 구조 설계, 현장 조사(노인 인터뷰) | 화면 설계서, DB 설계서 |
| 2단계: 핵심 기능 개발 | 3~6주 | 동네 지식 등록, 근거리 매칭 로직, 음성 입력 | 노인용 앱 MVP |
| 3단계: 연동·고도화 | 7~9주 | 카카오 알림 연동, 소모임 자동 제안, UI 개선 | 전체 서비스 통합 |
| 4단계: 테스트·실증 | 10~12주 | 노인 사용자 5~10명 UX 테스트, 복지관 연계 시범 운영, 오류 수정 | 시범적용 보고서 |

## 노인 친화 UX 설계 원칙 (확정 — 모든 화면에 적용)

1. 글자 크기 최소 20pt 이상 유지
2. 화면당 버튼 최대 3개 원칙
3. 텍스트 입력 최소화 → 선택지, 음성, 사진으로 대체
4. 전화번호 인증만으로 로그인 (복잡한 회원가입 없음)
5. 뒤로가기 버튼 항상 동일 위치 고정
6. 실제 노인 5명 대상 프로토타입 사용성 테스트 → UX 개선에 반영

## 글로벌 실증 계획

- **국내**: 복지관 연계, 독거노인·새 입주민 각 10~20명, 2~4주 시범 운영
- **일본(오사카)**: 사키시마 ATC 에이지리스센터, 오사카대 FICCT 협력, 노인 안내자 10~20명 모집

## 목표 지표 (실증 기준)

| 지표 | 목표치 |
|---|---|
| 동행 매칭 성사율 | 60% 이상 |
| 소모임 전환율 | 동행 3회 이상 쌍의 40% |
| 노인 이용자 만족도 | 4점 이상 / 5점 만점 |
| 사회적 고립도 변화 | 실증 전 대비 10% 이상 감소 |
| 한·일 실증 참여 인원 | 국내 20명 + 일본 20명 |

## 용어 정의 (도메인 모델용)

- **안내자**: 동네 지식을 등록하고 동행을 제공하는 노인 사용자
- **탐방자**: 새 입주민 또는 관광객, 안내자와 매칭되어 동행을 받는 사용자
- **동행**: 안내자와 탐방자가 실제로 만나 함께 걷는 활동 단위
- **동네 지식**: 안내자가 음성/사진으로 등록하는 지역 정보(가게, 산책로, 행사 등)
- **소모임**: 동행이 2~3회 반복된 쌍에게 자동 제안되는 정기 모임
